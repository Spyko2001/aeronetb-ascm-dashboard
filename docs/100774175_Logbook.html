2. Refined Logical Models

The first-semester design was continued and refined into a deployable hybrid model. PostgreSQL now stores the structured domains and MongoDB stores flexible document domains. The backend is responsible for validating cross-database references and combining data for the dashboard.

2.1 Refinements From Task 1

RefinementImplementation
Explicit RBAC modelAdded relational roles, permissions, role_permissions, and employee_roles tables, with API checks for every protected endpoint.
Versioned QC reportsNew QC submissions create a new MongoDB document version for the same order, part, supplier, and inspection type.
Immutable certificationsThe certification update route rejects changes after approval by checking is_immutable.
Audit coverageLogin, view, create, update, approve, export, and flag actions create audit entries with emp_id.
Deployment readinessAdded Render Blueprint, environment variables, automatic schema creation, automatic seeding, and a local memory fallback.

6. Security and Compliance

ControlImplementation
AuthenticationUsers log in with auth ID and password. Stored passwords are PBKDF2 hashes with salts.
Session protectionThe API issues signed HMAC session tokens with expiry.
RBACEach endpoint checks permissions in the backend before reading or mutating data.
Auditor restrictionThe auditor can read compliance data and raise flags but cannot mutate suppliers, orders, QC reports, or certifications.
Certification immutabilityApproved certifications have is_immutable=true. Later update attempts return HTTP 409.
Audit trailImportant actions write a record to audit_logs with emp_id, action type, entity, timestamp, outcome, and details.

8. Deployment Explanation

Render was chosen because it can deploy the Node.js service directly from GitHub and can create a PostgreSQL database from the same Blueprint file. MongoDB Atlas was chosen for document storage because the scenario includes variable QC payloads, certification evidence, and IoT readings. The Render service uses environment variables for all private connection strings. The same codebase runs locally in memory mode when database credentials are not present, which supports offline demonstration and automated testing.

9. Demo Flow

  1. Open the Render URL and show /api/health.
  2. Log in as Supply Chain Manager and show KPIs, alerts, supplier chart, QC chart, search, and CSV export.
  3. Log in as Procurement Officer and create a supplier or order.
  4. Log in as Quality Inspector and create/approve a QC report, then create and lock a certification.
  5. Try to edit the locked certification and show the immutability error.
  6. Log in as Equipment Engineer and add an IoT reading.
  7. Log in as Auditor and show read-only compliance access plus compliance flag creation.
  8. Show the audit log proving actions are attributed to emp_id.
